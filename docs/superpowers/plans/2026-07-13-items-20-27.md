> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Repair the duplicate SQLAlchemy table registration and deliver the verified cutover, governed workflow, API/CLI, Control Room shell, and Builder experience through Item 27.

**Architecture:** Domain modules own deterministic state transitions and invariants. Application services coordinate typed ports, while platform persistence, FastAPI, Typer, and React remain replaceable adapters. Every plan checkbox closes only after its focused harness and the full repository verification path pass.

**Tech Stack:** Python 3.13, SQLAlchemy 2, PostgreSQL 16, FastAPI, Pydantic 2, Typer, HTTPX, React 19, TanStack Query, TypeScript 5.9, Vitest.

## Global Constraints

- Never use `extend_existing=True` to mask duplicate table definitions.
- No business logic in HTTP handlers, CLI commands, or React components.
- Every authoritative mutation requires scope, exact source revision, local version fencing, approval, and writer ownership.
- Safety/completion semantics cannot be weakened by proposal, override, or break-glass.
- Every mutable workflow emits an immutable audit-facing result value.
- Items 20-27 remain open until static, unit, integration, registry, and frontend checks pass.
