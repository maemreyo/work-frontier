# Items 14-19 Tracer Bullet Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Deliver the first atomic persisted frontier, scoped identity controls, frozen #539 corpus, deterministic connection adapters, Level-3 GitHub boundary, and convergent ingestion pipeline.

**Architecture:** Keep solving and authorization pure, with Application-owned ports. PostgreSQL adapters persist one revision-fenced decision cycle transaction; GitHub and fixture adapters remain isolated behind typed connection contracts. Exact registered harnesses close Items 14-19 only after real PostgreSQL, MinIO, and GitHub sandbox gates pass.

**Tech Stack:** Python 3.13, SQLAlchemy 2 async, PostgreSQL 16 forced RLS, Alembic, cryptography AES-GCM/RS256, pytest/Hypothesis, Ruff, basedpyright.

## Global Constraints

- No mutable DecisionRecord updates or mixed-cycle frontier publication.
- No bare resource-ID repository operation without tenant/workspace scope.
- No plaintext session, MFA, OAuth, GitHub, or connection credential persistence.
- No arbitrary in-process plugin loading.
- No webhook authority without signature validation and authoritative refetch.
- No checkbox closure without exact registered harness evidence.
