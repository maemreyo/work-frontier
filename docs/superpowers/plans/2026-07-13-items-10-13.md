# Items 10-13 Implementation Plan

**Goal:** Complete the deterministic engine and the first production Platform wave without weakening the existing revision-bound evidence lifecycle.

**Architecture:** Item 10 remains pure Domain code. Items 11-13 live under Platform, use SQLAlchemy 2 metadata and async transaction-scoped repositories, and are enforced by PostgreSQL RLS, immutable audit triggers, MinIO content addressing, and PostgreSQL lease/fencing primitives. Registry-owned harnesses remain the only certification surface.

## Item 10 — Deterministic frontier engine

- Add immutable identified engine inputs and atomic DecisionRecord-set outputs.
- Implement readiness filters and configurable lexicographic ranking.
- Record item-local comparator traces and deterministic Recommended Next.
- Prove golden hashing, replay, ordering invariance, and blocker monotonicity over 10,000 inputs.

## Item 11 — Tenant-scoped persistence and RLS

- Define the complete SQLAlchemy metadata baseline with composite workspace keys.
- Add async engine/session utilities that set the non-BYPASSRLS app role and transaction-local tenant/workspace context.
- Add scoped repository contracts that cannot accept bare resource IDs.
- Add an Alembic migration that enables and forces RLS for every workspace table.
- Run direct-SQL, missing-context, cross-workspace read/write, and role-capability attacks.

## Item 12 — Immutable audit/evidence storage

- Implement canonical per-workspace SHA-256 audit segments with zero genesis.
- Add external signed-anchor verification and whole-segment retention proofs.
- Reject updates/deletes to append-only database tables through PostgreSQL triggers.
- Store bulky evidence under tenant/workspace/content-hash MinIO keys and verify round trips.

## Item 13 — Durable queue and scheduler fencing

- Implement durable inbox-before-ack and workspace idempotency.
- Claim jobs with `FOR UPDATE SKIP LOCKED` and lease-owner CAS.
- Add heartbeat, bounded deterministic retry, dead-letter, controlled replay, and fair selection semantics.
- Implement transactional outbox and scheduler lease overlap prevention.
- Inject rollback failures across inbox/snapshot/decision/projection/audit/outbox boundaries.

## Completion gate

- Run Ruff, formatter, basedpyright, all Python/frontend tests, migration/storage smoke, and every newly implemented registry harness.
- Only then change Items 10-13 to `[x]`.
- Commit the full batch atomically and run exact-revision certification through Item 13 on a clean checkout.
