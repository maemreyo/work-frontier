# Security policy

## Supported versions

Work Frontier is pre-release. Security fixes are applied to the current `main` branch; historical
commits and unmerged branches are not supported release lines.

## Reporting a vulnerability

Do not open a public issue for a suspected vulnerability.

Use GitHub's private security advisory flow for this repository and include:

- affected commit SHA or branch;
- minimal reproduction or proof of concept;
- impact and required preconditions;
- whether secrets or tenant data may be exposed;
- suggested containment or remediation, if known.

Do not attach real credentials, private repository payloads, production evidence, or personal data.
Use synthetic fixtures and redact logs. The maintainer will acknowledge the report, assess severity,
coordinate a fix, and disclose only after containment.
